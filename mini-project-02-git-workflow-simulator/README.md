# Mini Project 02: Git Workflow Simulator

## Project Overview

This project demonstrates a professional Git and GitHub workflow used by software development teams.

The project simulates a 3-person team workflow using feature branches, pull requests, code reviews, merge conflict handling, and release tagging.

## Project Theme

The simulated project is called TeamFlow, a basic team task management system used only for demonstrating Git workflow.

## Objectives

- Use a single shared GitHub repository
- Create feature branches for separate work
- Push branches to GitHub
- Open pull requests
- Simulate code review
- Resolve merge conflicts
- Merge approved changes into main
- Create a release tag
- Document workflow in CONTRIBUTING.md
